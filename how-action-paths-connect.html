How action paths connect

Most CMR cases follow one of five action paths. Recognize the pattern, and the right tag, request action, and account action fall into place.

The five patterns

Tap each pattern to see the full action path, examples from SoAA, and any documentation notes.

PATTERN A

No account action

Tag the violation, but the user keeps their account.

Explore →
The path
1Tag violation
→
2Delete request
→
No account action
Examples in SoAA
Mismatched category Non-US service location Barter or free service Questionable name Test posts Not a service Seeking job or partnership Mild legally questionable Mild inappropriate language
PATTERN B

Hold the account

Pause activity for further investigation. No permanent action yet.

Explore →
The path
1Tag violation
→
2Delete request
→
3Hold the account
Examples in SoAA
Spammy customer (5+ signals, doesn't meet ghost criteria) Spammy pro (5+ signals, doesn't meet delete/blocklist criteria)
NoteThe Hold reason should match the investigation type — for spammy cases, "User being investigated — Deceptive Practices."
PATTERN C

Ghost the account

Make the account invisible to the platform without deleting it.

Explore →
The path
1Tag violation
→
2Ghost the account
Example in SoAA
Spammy customer (signals meeting ghost criteria, no delete/blocklist)
NoteGhost cases don't require deleting the request — the request is left in place while the account is hidden.
PATTERN D

Delete the account

Remove the account permanently, but no blocklist entry.

Explore →
The path
1Tag violation
→
2Delete request
→
3Delete the account
Example in SoAA
Minor user (13–17)
NoteDocument the exact age if known. If unknown, use the age range on the request.
PATTERN E

Delete and blocklist

Remove the account and add to the blocklist to prevent re-registration.

Explore →
The path
1Tag violation
→
2Delete request
→
3Delete account
→
4Add to blocklist
Examples in SoAA
Minor user (under 12) Sexual content Discrimination (slurs) Severe inappropriate content Severe legally questionable Competitor service Spammy pro (with strong evidence)
NoteBlocklist reason language varies by violation — see SoAA for the exact tag (Community Standards Violation, Scam/Fake Requests, Minor/Underage).
