The Standard Contact Verification Workflow

Three steps. Most of the work happens in Step 2.

1

Start in the Work Ticket

Open the Syndication Portal link from the Work Ticket Details section. This takes you into the portal where you'll do the review.

2

Review and decide in the Syndication Portal

The five substeps where most decisions get made.

Expand →

  1. Review the request for deceptive behavior and TOS violations. Use the Deceptive Practice Identification KA to determine whether the request shows signs of deceptive behavior or meets the criteria for a violation.

    Open Deceptive Practice Identification KA

  2. Check for strong links to banned accounts or pro accounts. Use the Linked Accounts Admin Page and the user's default phone number. Use the Linked Account Checking and Universal Actioning KA for guidance on identifying strong links and determining the correct mirrored action.

    Open Linked Account Checking & Universal Actioning KA

  3. If the user is strongly linked, mirror the banned account action. Match the action taken on the linked account so the same enforcement applies consistently.

  4. Choose the request action in the Syndication Portal. Based on what you found in steps 1–3, select the correct action for the request itself.

  5. Add notes when deleting the request — except in cases of sensitive violations like Sexual Content, Discrimination, or similar. For those, the violation tag is enough; no additional notes needed.

NoteIf you're unsure whether a link counts as "strong," the Linked Account KA above has worked examples. When in doubt, escalate before mirroring an action.
3

Finish the Work Ticket

Complete Outcome and Admin Actions. See the field map.

Expand →

Back in the Work Ticket, complete the Outcome and Admin Actions fields based on what you found in Step 2. The fields you'll touch are highlighted below.

WORK TICKET FIELDS

The three highlighted sections show where the work happens.
Top left — Details (Step 1): the Syndication Portal link you open to start the review.
Right — Outcome (Step 3): Routing Skill, Outcome, Outcome Detail, TOS Violations, Internal Notes.
Right — Admin Actions (Step 3): Account Actions, Fides Rules.
