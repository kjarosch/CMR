      The Standard Contact Verification Workflow

      ﻿

      In a standard Contact Verification case, follow this order:

      Start in the Work Ticket

      Open the link in Work Ticket Details.

      Review and decide in the Syndication Portal

      1. Review the request for deceptive behavior and/or TOS violations.

      Use the TOS Violations Overview KA to confirm whether what you find meets the criteria for a violation. Use the Deceptive Practice Identification KA (000013733) to determine whether the request shows signs of deceptive behavior. 2. Check for strong links to banned accounts or pro accounts using the Linked Accounts Admin Page and the user's default phone number. Use the Linked Account Checking and Universal Actioning KA for guidance on identifying strong links and determining the correct mirrored action.

      3. If the user is strongly linked, mirror the banned account action.

      4. Choose the request action in the Syndication Portal.

      5. Add notes when deleting the request except in cases of sensitive violations like Sexual, Discrimination, or the likes.

      Finish the Work Ticket

      Complete Outcome and Admin Actions.
